  • Company: Imperial College London
  • Project date: 2022
  • Project URL: https://github.com/kert114/Mars-Rover-Project

Project Details

As part of the second year summer project at Imperial College london me and my team of 6 others had to create an Autonomous Mars Rover. The rover would have to compete against other teams on the surface of "Mars" and try to locate all of the aliens (differently coloured ping pong balls) with its on board vision system and detect their underground power stations with the radar as accurately as possible. The locations of the aliens and their stations would be mapped live to a website also showcasing the path the rover took to find the aliens. In the end the rover returns to its starting position and shuts off. The rover can be broken down to 7 subsystems that were all vital for the success of the rover. I was responsible for the Control, Command and Drive subsystems.
  • Command subsystem - A website that would display the live location of the rover, the detected aliens and their bases on the arena. The website also displays the battery level and allows the operators to take control of the Autonomous rover. The website was developed using JavaScript and the connection was established using a TCP connection.
  • Drive - responsible for the navigation around the arena using an optical sensor mounted at the back of the rover. The rover is autonomous so a path finding algorithm had to be implemented alongside the manual control to allow the operator to intervene when problems might occur.
  • Control - The control module handles the communication between all the other subsystems and has logic built in C++ to prioritize actions over each other and controls the rover based on the different inputs. The logic of control is handled on an ESP-32 micro controller. The control module uses SPI and TCP communication methods to talk to other subsystem.
